4章: 関数定義

関数を定義する

関数は，型の制約と処理の内容を二つ重ねて書くのが一般的みたいです[要出典]．たとえば，

is_even :: Integral a => a -> Bool
is_even n = n `mod` 2 == 0

のように．

ただ，Haskellは型推論が強力なので，たとえば

is_even_s n = n `mod` 2 == 0

といきなり書いてしまっても

*Main> :t is_even_s
is_even_s :: Integral a => a -> Bool

と全て察した答えが返ってきます．えらいね．

条件式とガード

たとえば，t >= 0のときに1を返す関数

int unit_step(int t) {
    // 普通は return t >= 0 ? 1 : 0; とか書くのかもしれない
    if (t >= 0)
        return 1;
    else
        return 0;
}

をHaskellではどう書くか．

まず，Haskellにもifがあるので，それを使ってみましょう．

というわけで，Haskell版 unit_step．

unit_step :: Int -> Int
unit_step t = if t >= 0 then 1 else 0

はい，見た通りですね．

Haskellでは，if-then-else の else が必須となります．省略不可です． そりゃあまぁ，elseが何も返さない時にifが返す値が無くなるので当たり前っちゃ当たり前の話ですか．

*Main> unit_step (-1)
0
*Main> unit_step 4
1

はい，ありがとうございます．

じゃあ，今度は t < 0 ,t > 10 のときに1を返す関数について． 先ほどのように素直に

zttz :: Int -> Int
zttz t = if t < 0 then 1 else if t > 10 then 1 else 0

と書いてもいいのですが，可読性がいまいちです．そんな時は，ガード付きの等式というものがあります．

zttz  n | n < 0 = 1
        | n > 10 = 1
        | otherwise = 0

みたいなね．ちなみに，| の位置はそろえる必要があるみたいです． vimでgg=G すると全部行頭に来ちゃって困るね．仕方ないね．

パターンマッチ

さっきから

unit_step :: Int -> Int
unit_step t = if t >= 0 then 1 else 0

みたいな書き方をしていますが，もしも

pulse :: Int -> Int
pulse t = if t == 0 then 1 else 0

のような場合，もうちょっと直観的に書ける方法があります．そう，パターンマッチです．

pulse :: Int -> Int
pulse 0 = 1
pulse _ = 0

こんな感じ．_は何でも受け取ります．

さて，このパターンマッチ，何が嬉しいのかというと．

いろいろなパターンマッチ

たとえば タプル．coupleから2つめの値を取り出そうとするとき

snd :: (a, b) -> b
snd (_, y) = y

のように，タプルに対してマッチングさせることができます．

たとえば リスト．リストの2番目の要素を取得してくるような，そんな 見本誌を避けて取るような動作をさせたいときには

pick_usuihon :: [b] -> b
pick_usuihon [_, b, _, _ ] = b

のようになります．ただしこれには問題があって，リストが4つの場合しか受け取れません． こんな時には， cons演算子 : を使います．

リストは[1, 2, 3, 4] のように見えていますが，実は(1 :(2 :(3 :([])))) みたいな感じの表現に なっています．なので，

pick_usuihon (_:b:_) = b

と書けば，後ろの _ に残りのすべての部分がキャプチャされるので，

*Main> pick_usuihon [1,2,3,4,5]
2

と，見事2冊目をゲットできるわけです．

ラムダ式

Haskellはもちろん[要出典]，ラムダ式を定義することができます．

\x -> x + x

以上です．味気なし．よく使うし，あえて紹介するまでもないかもしれないけど

*Main> let power = \x n -> x ^ n
*Main> power 2 5
32

みたいな感じに遊べます．

セクション

1 + 2 の + みたいな演算子は，(+) みたいにすることでカリー化された関数になるらしいです．

*Main> (+) 5 10
15

おお，すごいすごい，

で，このカリー化された関数 (+) とか，それを部分適用してあるやつ (x +) (+ y) っていうのを セクションって呼ぶらしいです．

ちょっとこのあたりあんまり重要度が分かってないので，まぁ，おいおい分かるのでしょう．

まとめ

関数型言語の関数についてさらーっとやりました．「関数」について分かったので，残すは「型言語」だけですね．

3章: 型とクラス

1章2章は環境構築や基本部分なので飛ばしちゃいました．

基本型とリスト，タプルが主な型

基本型としては以下の型が例にあげられてました．

  • Bool – 真偽値
  • Char – 文字
  • String – 文字列
  • Int – 固定精度倍数
  • Integer – 多倍長整数
  • Float – 単精度浮動小数点数

ざっと調べたところ，小数の型にはいろいろあるみたい．追々分かるのかな．

リストとかタプルは

['リ','ス','ト'] :: [Char]
('タ','プ','ル') :: (Char, Char, Char)

この通り．

全ての式は型を持つ

静的型付け言語であればだいたいなんでもそんな感じですよね．
ただ，Haskellにおいては型推論が強力で，割と何も言わなくても綺麗に型が付くみたいです．

C#で

(x, y) => x + y

とか書くとコンパイラにぶちのめされますが，Haskellなら

\x y -> x + y

でも怒られません．

Prelude> :type \x y -> x + y
\x y -> x + y :: Num a => a -> a -> a

みたいにちゃんとNumを二つとってNumを返す感じになってます．

ちなみにNum a => a -> a -> a部分の見方は，Numクラスのインスタンスである型aについて型aの値を2回取って型aの値を返す，みたいな感じらしいです．

型クラスっていうのは要するにinterface みたいなものでしょうか． Numにキャスト可能な型aにおいてaをとってaを取ってaを返す，みたいな？

実際にはNumにはさっきのIntやらIntegerやらFloatやらが代入可能みたいです．

カリー化

どんな料理にもカレー粉をぶち込むとカレー味に染まるアレではないです．

多引数関数を1引数関数の高階関数に変換するアレです．

\x y z -> x + y + z

が

\x -> (\y -> (\z -> x + y + z))

になるというおはなし．

実は + も Num a を二回取ってaを返すタイプの関数なので

Prelude> :type (+)
(+) :: Num a => a -> a -> a

みたいな感じです．

そしてそして，カリー化された関数は部分適用というワザが使えまして，

Prelude> :type (+ 2)
(+ 2) :: Num a => a -> a

のように (+ 2) が Num aを取って2を足して返す関数になってるわけです．

Prelude> let plus2 = (+ 2)
Prelude> plus2 5
7

ね．(ちなみにghciにおいて，letを使うと変数を束縛できるらしいです)
